# UX Research and Strategy

A foundational skill for defining the logical flow and user-centric architecture of a product before any pixels are styled.

## 1. Empathy & Discovery
Before proposing a layout, you must define WHO is using the product and WHAT their primary objective is.
- Outline 1-2 core user personas. What are their pain points?
- Define the "Happy Path" (the primary, ideal user flow to conversion/success).

## 2. Information Architecture (IA)
Organize the content logically.
- What information needs the highest visual priority?
- Group related actions and information to reduce cognitive load (Miller's Law - keep choices around 7±2 where possible).
- Propose a clear navigation hierarchy (e.g., Top Nav vs. Sidebar vs. Tab Bar).

## 3. Wireframing & Flow Logic
When providing wireframe concepts (in markdown, text, or pseudo-code), focus purely on structure, not styling.
- Clearly delineate layout sections: Header, Hero, Content Grid, Sidebar, Footer.
- Define what happens on primary interactions (e.g., "Clicking X opens a modal containing Y").

## Output Format
When delivering UX strategy, structure your response as follows:
### User Persona Context
[Brief description of the intended user]
### Journey Map
[Step-by-step flow from entry to goal completion]
### Information Architecture / Wireframe Logic
[Structured breakdown of the page or app layout]

**Remember**: A beautiful UI cannot save a confusing UX. Establish the flow first.
